# VPS_DEPLOYMENT.md

- **Existencia en VPS:** Probablemente sí, para soporte remoto (soporte_remoto.sh, ssh_cmd.py, deploy_bridge.py indican interacciones con VPS o equipos distribuidos).
- **Dominio/Endpoint:** Por confirmar.
- **Gestión:** No se deben ejecutar comandos como docker system prune o 
m -rf sin autorización.
- **Diferencia con Render:** Render parece el backend cloud (API/Admin), mientras que local/VPS puede referirse a los terminales TPV físicos (Raspberry/Windows).
